# Target Personas - WBcom Essential

---

## Persona 1: Community Builder Chris

### Demographics
- **Role:** Online community manager / Membership site owner
- **Age:** 32-45
- **Team Size:** Solo or small team (1-3)
- **Industry:** Education, Coaching, Niche Communities
- **Technical Level:** Intermediate (comfortable with WordPress, not a developer)

### Goals
- Build an engaged online community without hiring developers
- Display members and groups in attractive layouts
- Create a professional-looking community site that rivals expensive platforms
- Reduce dependency on multiple plugins

### Pain Points
- BuddyPress out-of-the-box looks dated and bland
- Existing solutions require custom CSS or expensive page builders
- Elementor widgets don't play well with BuddyPress data
- Too many plugins cause conflicts and slow sites
- Members complain the site looks "amateur"

### Why WBcom Essential Appeals
- 11 dedicated BuddyPress blocks that actually understand BP data
- Members Grid, Groups Carousel work without custom code
- Profile Completion block encourages engagement
- Theme Colors toggle means everything matches automatically

### Key Message
> "Turn your BuddyPress community from basic to beautiful without touching code."

### Content They Respond To
- Before/after screenshots
- "How to build X in 10 minutes" tutorials
- Community site examples
- Member engagement tips

## Persona 2: Developer Dave

### Demographics
- **Role:** Freelance WordPress developer / Agency developer
- **Age:** 28-40
- **Team Size:** Solo or agency (2-15)
- **Industry:** Web Development, Digital Agency
- **Technical Level:** Advanced (builds custom themes, knows PHP/JS)

### Goals
- Deliver client projects faster without reinventing common components
- Maintain consistent quality across projects
- Reduce time spent building carousels, sliders, pricing tables from scratch
- Find reliable, well-coded plugins that don't create technical debt

### Pain Points
- Building the same components repeatedly wastes billable time
- Client wants "that slider effect" but budget doesn't cover custom dev
- Many block plugins have bloated code or poor performance
- Elementor widgets don't always match theme styling
- Needs both Elementor AND Gutenberg options for different clients

### Why WBcom Essential Appeals
- 45 production-ready blocks covering common needs
- Theme Colors toggle inherits client's theme automatically
- Clean code structure (no jQuery in blocks, modern React)
- Works with Elementor OR Gutenberg based on client preference
- CSS custom properties for easy customization

### Key Message
> "Stop rebuilding sliders and pricing tables. Start delivering projects faster."

### Content They Respond To
- Code quality details (build system, no jQuery, etc.)
- Performance benchmarks
- Integration documentation
- "How it works under the hood" technical posts
